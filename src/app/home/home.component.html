Disclaimer:

As I'm only using a free-tier subscription to https://www.football-data.org/, the website can only handle a limited number requests per minute, thus you may encounter error messages if you browse through the site too fast.

This site is merely a personal project for me to gain experience in using Angular and by no means meant to provide completely accurate and up-to-date information on the Premier League.


Features

  1. General
    • HTTP Service
      • Centralised service to make HTTP requests to https://www.football-data.org/ API
      • Created service to managed error messages
      • Spinner graphic shown while data is being fetched
    • Custom Attribute Directives
      • Highlight directive that will add a class to the element when mouse hovers over
    • Animations
        • Simple animations to give smoother transitions between components
    • Responsiveness
      • The components of this SPA are designed to be readable in both mobile and desktop versions
      • Largely composed of ng-Bootstrap and Angular Material components with flex layout and Bootstrap Grid layout
    • Design
      • The main focus of the SPA was to utilise the various functionalities/tools of Angular
      • The aesthetic features of the site are therefore generally lacking
  2. Home
    • Reactive Form
      • Simple Reactive feedback form with relevant validators
      • Validation error messages appear below field
      • (Submitting form doesn't send the information anywhere)
  3. Nav-bar
    • Responsive Angular Material nav-bar turns into side-nav for smaller screens
    • Provides router links to all the various components
    • Login and Register feature with connection to Firebase for authentication
  4. Standings
    • ng-Bootstrap Carousel component used to display images/position of top teams
    • The league table fetched from the API is up-to-date and provides links to individual team details
  5. Schedule
    • Holds the Top Scorer, Schedule and Results components
    • Top scorer table request is sent first to obtain current matchday and top scorer information
    • Schedule and Results components take the current matchday information as input to process requests
    • View more matches button at the bottom will load the future/past match data
    • Angular Material tabs component used to toggle between schedule/results
    • Links to individual teams provided
  6. Teams
    • Mat-card module used to format the team cards
    • Links to individual team page via "/teamdetails/:id"
  7. Individual Team Details
    • ng-Bootstrap accordion used to display players (grouped by position)
    • Clicking on player names links to modal which displays more information on the player
  8. User Page
    • User landing page which can only be accessed if logged in with valid account
    • Redirects to home page if user not logged in
  9. Possible extensions
    • Footer
    • Search bar
